# Qodo Merge (PR-Agent) 動作テスト用プロジェクト

このプロジェクトは、AIによるプルリクエストレビューツール **Qodo Merge (PR-Agent)** の機能を評価するためのテスト用リポジトリです。
意図的に不適切な実装（バグやアンチパターン）を組み込んでおり、AIがこれらをどこまで正確に指摘できるかを確認することを目的としています。

## 意図的に組み込まれた問題点

本プロジェクトの `feature/test-review` ブランチには、以下の問題が意図的に仕込まれています。

### バックエンド (Laravel/PHP)
- **SQLインジェクションの脆弱性**: `UserController` で生クエリに外部からの変数を直接埋め込んでいます。
- **N+1問題**: `PostController` でリレーション（User, Comments）を `with()` せずに取得し、ループ内でのクエリ発行を誘発させています。
- **トランザクション不足**: 複数テーブル（Posts, Tags）を更新する際、エラー時の整合性を保つためのトランザクション処理を行っていません。
- **Null安全の欠如**: `find()` の結果が `null` である可能性を考慮せず、直接プロパティにアクセスしています。
- **バリデーションの配置不備**: `FormRequest` を使わず、コントローラ内にバリデーションロジックを直書きしています。

### フロントエンド (React/TypeScript)
- **型の安全性欠如**: `any` 型を多用し、TypeScriptのメリットを打ち消しています。
- **Propsの型定義なし**: コンポーネントのPropsに型定義を行っていません。
- **useEffectのクリーンアップ不足**: 非同期処理（fetch）のキャンセル処理がなく、メモリリークの可能性があります。
- **マジックナンバーの使用**: 数値を定数化せず、ロジック内に直接書き込んでいます。
- **APIレスポンスの型未定義**: APIから取得したデータに対して型定義を行っていません。

## テスト方法
1. `main` ブランチに対して `feature/test-review` ブランチからプルリクエストを作成します。
2. Qodo Merge (PR-Agent) が実行され、上記の項目についてレビューコメントを出すか確認します。
3. （`.pr_agent.toml` の設定により）日本語とベトナム語の両方でコメントが生成されることを確認します。
